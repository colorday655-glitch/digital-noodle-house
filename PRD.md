# 无人面馆应用 - 极简需求文档（PRD）

## 1. 产品定位
- **产品名称**：张大爷无人面馆
- **目标用户**：村里老年人（尤其是张大爷）
- **核心价值**：操作极简、出餐超时有补偿

---

## 2. 功能需求

### 2.1 菜品列表
| 菜品 | 价格 |
|------|------|
| 红烧肉面 | ¥15 |
| 清炒时蔬 | ¥8 |

### 2.2 下单流程
- **零门槛**：无需注册、无需手机号验证
- 首页直接显示两个菜品按钮，点击即下单
- 支付方式：仅支持现金投币（实体机）或村委代付

### 2.3 超时补偿逻辑
- 出餐时间上限：**15 分钟**
- 超时判定：点击下单后 15 分钟内未完成出餐
- 补偿方式：自动弹出红包（金额随机 ¥1-3）
- 红包领取：出示给村委工作人员兑换

---

## 3. 适老化 UI 设计规范

| 元素 | 尺寸要求 |
|------|----------|
| 按钮高度 | ≥ 80px |
| 文字大小 | ≥ 32px |
| 菜品图片 | 清晰实拍，避免抽象 |
| 颜色对比 | 背景与文字对比度 ≥ 4.5:1 |
| 操作步骤 | ≤ 3 步 |

---

## 4. 页面结构（仅 2 页）

1. **首页**：两个巨大的菜品按钮 + 倒计时显示
2. **出餐页**：显示"请稍候" + 预计剩余时间

---

## 5. 非功能性要求
- **响应速度**：页面加载 ≤ 1 秒
- **离线可用**：断网时仍可投币下单
- **故障反馈**：机器异常时显示超大字体提示"请联系村委王大爷 138-xxxx-xxxx"
